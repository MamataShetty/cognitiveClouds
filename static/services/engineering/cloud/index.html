Build Your Private Cloud

OpenStack is our favorite open source library for creating private and public clouds. We’ve also written a few tools and libraries that help us create redundancies across Amazon, Rackspace, and VMWare clouds.

  • OpenStack Nova

    This is the main computing component, managing the virtualization hosts, the network between them and all existing virtual machines..

  • OpenStack Glance

    Thanks to this service, it's easy for you and your users to start new virtual machines. Pre-built operating system images can be stored within Glance and are then available on demand.

  • OpenStack Swift

    While many consider the cloud to be about virtualization only, also storage is an important aspect of the cloud principle. The Swift object store allows for easy and scalable file storage within the cloud.

  • OpenStack Horizon

    This is OpenStack's web interface (dashboard) that allows you to easily manage all aspects of your cloud like user management (via Keystone) or VM management.


Planning

Migrating an entire set of systems, processes and applications to the OpenStack Cloud can be a time-consuming process that involves consolidation and standardization.

Use cases

A staged approach to leveraging the Cloud starts with identification of the set of use cases that can become part of the initial roll-out. Some of the common use cases that we see as part of the initial roll-out are:

  1. Dev/Test Cloud – makes it easy to provision and de-provision resources faster and easier, and hence facilitates business agility.
  2. Storage – the advent of big data has many customers looking to the Public Cloud for their storage and related analytics needs.
  3. Cloud Bursting – makes it easy for enterprises such as Retail, with spiky resource needs, a mechanism to leverage the Cloud whenever resource usage exceeds a certain threshold.

Current Assets

What current under-utilized assets, hardware and software, can be re-purposed for a Private Cloud? Indeed, proper utilization of existing assets is perhaps one of the most important drivers for adoption of a Private Cloud.

Roadmap

Adoption of the Cloud has to be planned such that it doesn’t negatively impact current operations. Resource identification and allocation, provisioning of temporary systems and support resources during transition, project schedule, validation and testing, training and finally switchover have to be addressed. Overall Cloud roadmap, with emphasis on earlier stages, is also recommended.


Architecture + Implementation

Your private cloud architecture is driven by hardware selection, vendor selection, location, sizing, redundancy, service levels and disaster recovery requirements.

Cloud Selection

Not all public clouds and private clouds are created equal. On the Public Cloud side, Amazon Web Services (AWS) has the best set of complementary platform services in support of their infrastructure services. Joyent Cloud claims a second-to-none high performance cloud based on the open source SmartOS. Even among OpenStack Public Cloud providers, different providers highlight different features: HP Cloud through a broad set of eco-system solution partners and Rackspace a fanatical managed support. With private cloud software, apart from the paid software from VMWare, the most popular open source options – OpenStack, CloudStack and Eucalyptus – have different pros and cons. PistonCloud has a ruggedized version of OpenStack specifically geared towards enterprises. It is important to assess all the options and pick the option that most aligns with your needs.

PaaS

Most cloud adopters will need a set of platform services (Platform-as-a-Service) to complement the infrastructure services that are offered by OpenStack. Examples of these platform services include elastic load balancing, cloud monitoring, auto-scaling, application stack templates, caching and relational database service among others. A number of vendors such as Transcend Computing and Cloud Foundry offer these services. All relevant options an related alignment with organizational goals have to be assessed.

Tool Selection & Implementation

The major activities here include: hardware and tool acquisition, setup and integration. Systems, processes and applications have to be migrated and validated.
